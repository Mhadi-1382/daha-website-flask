    مستندات وب سرویس ها (APIs)

    وب سرویس یا API چیست؟

    API مخفف Application Programming Interface است. وب سرویس مجموعه ای از پروتکل ها، روندها و ابزارهایی است که به نرم افزارهای مختلف اجازه می دهد تا با یکدیگر ارتباط برقرار کنند. وب سرویس ها به عنوان پلی بین دو یا چند برنامه عمل می کنند و به آنها اجازه می دهند داده ها و عملکردهای خود را با یکدیگر به اشتراک بگذارند.

    سرویس API برای دسترسی به اطلاعات پلتفرم داها طراحی شده است تا توسعه دهندگان بتوانند به راحتی از بخش های مختلف پلتفرم استفاده کنند. با استفاده از وب سرویس ها می توانید عملکردهایی همچون دریافت اطلاعات دقیق کاربران، آگهی های ثبت شده در قسمت فن آور و... را در برنامه خود داشته باشید.

    نحوه متصل شدن به API

    بعد از ثبت نام و دریافت رمز عبور (برای احراز هویت) میتوانید به راحتی به وب سرویس ها متصل شوید، برای اتصال به وب سرویس شما به نام کاربری یا همان ایمیلی که هنگام ثبت نام وارد کردید و رمز عبوری که از طرف کارشناسان ما برای شما ایمیل میشود نیاز دارید.

    تست و مستند APIs Swagger

    نحوه تست در APIs Swagger

    نحوه اتصال در پایتون

    import requests as rq
from requests.auth import HTTPBasicAuth

USERNAME = ""
PASSWORD = ""

URL = "https://dahauni.ir/api/APINAME"

response = rq.get(URL, auth=HTTPBasicAuth(username=USERNAME, password=PASSWORD))
print(response.json())

    پارامتر های ورودی

    • USERNAME: در این متغییر باید آدرس ایمیل خود که هنگام ثبت نام وارد کردید را قرار دهید.
    • PASSWORD: در این متغییر باید رمز عبوری که برای شما ایمیل شده است را قرار دهید.
    • URL: در این متغییر باید آدرس وب سرویس مورد نظر خود را قرار دهید، البته آدرس https://dahauni.ir/api/ ثابت می باشد و شما فقط باید APINAME یا نام وب سرویس را وارد کنید، برای دریافت نام یا دسته بندی وب سرویس ها میتوانید از تست و مستند APIs Swagger استفاده کنید.

    لیست کدها و خطاها

    با فراخونی API احتمال دارد شما با کد ها و خطاهای زیر روبه رو شوید. این کد ها عبارتند از:

    • 200: نشان دهنده موفقیت آمیز بودن اتصال شما به API می باشد.
    • 404 یا Not Found: نشان دهنده این است که آدرس مورد نظر وجود ندارد، که احتمال دارد آدرس API را اشتباه وارد کرده باشید.
    • Authentication failed: نشان دهنده مشکل در اتصال شما به API، که در اثر نادرست بودن نام کاربری یا رمز عبور می باشد.

    قوانین و مقرارت استفاده از APIs

    توسعه دهنده عزیز، بعد از ثبت نام در توسعه دهندگان داها شما بصورت رسمی مجوز استفاده از اطلاعات دیتابیس (بصورت محدود) را خواهید داشت، بر این اساس شما باید تابع قوانین تعریف شده باشید، قوانین عبارتند از:

    • توسعه دهنده اجازه ندارد از اطلاعات در جهت تبلیغات، سوء استفاده، فروش اطلاعات، تهدید، نفوذ، هک و... استفاده نمایید، اگر داها متوجه این تخلفات شود با فرد خاطی برخورد و حساب او را نیز مسدود می کند.
    • استفاده از اطلاعات تنها با ذکر منبع (نام داها) امکان پذیر است.
    • تلاش برای نفوذ و استخراج اطلاعات و استفاده از هر گونه ربات جهت دسترسی خودکار به سایت و پایگاه داده های داها و برداشت اطلاعات از آن ممنوع است.
    • تلاش برای رمزگشایی و کنکاش از کدهای منبع داها ممنوع است.
    • از کار انداختن و ایجاد هر گونه اختلال در سرویس های داها ممنوع است.
    • انتقال کدهای مخرب، ویروس ها، کرم های رایانه ای و... به داها ممنوع است.
    • حفظ حریم خصوصی کاربران ضروری است.
    • هیچ فرد حقیقی، حقوقی یا سازمانی مجاز به تغییر محتوای داها نیست.
    • اگر توسعه دهنده ای ایده ای جهت توسعه پلتفرم داها دارد حتما آن را با ما در میان بگذارد.
    • داها این حق را دارد تا در هر زمانی که نیاز بود، قوانین و مقرارت را تغییر دهد و توسعه دهندگان باید از قوانین جدید پیروی و آن را دنبال کنند.

      • استفاده از وب سرویس های داها (APIs Daha) به منزله مطالعه و پذیرش قوانین و مقرارت استفاده از خدمات داها است.
